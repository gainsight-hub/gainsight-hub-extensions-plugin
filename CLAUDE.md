# Gainsight Hub Extensions Plugin

## Eval Rules

- **Baseline isolation**: When running skill evals, baseline (without_skill) agents must run in an isolated git worktree with skill content (`skills/*/SKILL.md`), fixture widgets, test directories, and `*-workspace/` eval output directories removed before execution. Keep `skills/*/evals/` intact — the eval runner needs `evals.json` for test definitions, and these don't leak skill knowledge. This prevents baselines from learning framework patterns by reading repo files or prior eval outputs, which inflates their scores and compresses the skill delta.
- **Multi-trial scoring**: Run each eval 3 times per configuration (with_skill, without_skill). Save to `<eval-name>/<config>/run-1/`, `run-2/`, `run-3/` subdirectories. Report mean ± stddev. The `aggregate_benchmark.py` script expects this layout.
- **Eval viewer `</script>` bug**: The skill-creator's `generate_review.py` embeds output file contents inside a `<script>` block via `json.dumps`, but doesn't escape `</script>` in the JSON. Widget `content.html` files contain `<script>` tags which break the viewer. **Workaround**: After generating the static HTML, post-process it to replace `</script>` with `<\/script>` inside the `EMBEDDED_DATA` assignment. Alternatively, don't copy `content.html` into `outputs/` (keep it in `outputs/dist/` where the viewer doesn't recurse). We don't own `generate_review.py` — don't modify it.
- **Eval viewer directory conventions**: The viewer discovers runs by finding directories with an `outputs/` subdirectory, but only reads immediate children of `outputs/` (not recursive). Place `eval_metadata.json` in the config directory (e.g., `polished-card/with_skill/`) so the viewer finds prompts (it checks `run_dir.parent`). Copy key files like `response.md` into `outputs/` so the viewer displays them.
- **Multi-turn evals**: Evals with a `turns` array in evals.json test the full conversational workflow (ask questions → receive answers → build). Use `scripts/run-multi-turn-eval.py` which runs `claude -p` for turn 1 then `claude --resume` for follow-ups. Single-turn evals (no `turns` field) can still be run via subagents as before. The runner saves transcript.json, response.md, timing.json, and eval_metadata.json automatically.
